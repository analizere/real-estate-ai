# Phase 2A — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Phase UI Surface

Phase 2A is primarily a services/infrastructure phase. There are no new pages. The UI surface
is limited to three new UI elements added to existing pages:

1. **UsageMeter component** — added to `app/account/settings/` (D-20)
2. **Inline usage counters** — added inline on the address search bar and skip trace feature (D-21)
3. **80% warning banner** — surfaces both inline on the relevant feature and in Account Settings (D-18)
4. **100% block message** — contextual inline message on the specific exhausted action only (D-19)

All other work in this phase is service-layer and has no UI surface.

## Interaction Contract

### UsageMeter component

- Renders in Account Settings page below the SubscriptionStatusCard, above the Profile section
- Fetches usage data via React Query (per CLAUDE.md mandate) from `/api/v1/usage`
- Shows Skeleton placeholder while loading
- Updates optimistically when a metered action completes (usage count increments immediately in UI)
- On rollback (action failed server-side), counter reverts without toast — silent rollback

### Inline Usage Counter (address bar)

- Renders as a Badge (`variant="secondary"`) appended to the right side of the address input
- Format: "{used}/{limit}" (e.g. "47/50") or "Unlimited" when limit is 'unlimited'
- Color: `text-muted-foreground` at 0–79%, `text-yellow-600` at 80–99%, `text-destructive` at 100%
- Does NOT block the input field at 100% — the block message appears below the input only

### 80% Warning Banner

- Position: immediately below the relevant input/feature that is approaching its limit
- Also appears in Account Settings usage section as an inline note beside the relevant row
- Dismissible: no — it persists until usage drops below 80% or month resets
- Icon: Lucide `AlertTriangle` in `text-yellow-600`

### 100% Block Message

- Position: replaces the feature's primary action button with the block message
- It does NOT disable the whole page or show a modal
- The exhausted action is the only thing blocked — all other features remain fully interactive
- Icon: Lucide `Ban` in `text-destructive`
- "Upgrade to Pro" button uses `variant="default"` (primary, black)
- "View Usage" is a plain text link (`text-sm text-muted-foreground underline`)
