All pages are `"use client"` components. All use the AuthCard wrapper, shadcn components, and follow these UI-SPEC Interaction Contract rules:
- Validate on submit (not every keystroke)
- Show field-level error below the input with `role="alert"` (via FormField component)
- Clear field error on next input change
- Disable CTA button and show Spinner (sm, 16px, inside button) during async operations
- Do NOT disable or clear form inputs during loading

**1. `app/(auth)/sign-up/page.tsx`** — Sign Up page:
   - Display headline above the card: `<h1 className="text-[28px] font-semibold leading-[1.15] text-center mb-6">Analyze any deal in 60 seconds.</h1>` (Display typography from UI-SPEC, focal point for sign-up only)
   - AuthCard with title "Create your account"
   - Form fields: Name (text input), Email (email input), Password (PasswordInput component)
   - Primary CTA: Button text "Create account" (per UI-SPEC Copywriting)
   - Below CTA: Separator with "Or continue with" text
   - GoogleOAuthButton
   - Footer link: "Already have an account? Sign in" linking to `/sign-in` (per Copywriting Contract)
   - On submit: call `signUp.email({ email, password, name })` from auth-client
   - On success: redirect to `/verify-email?email={email}` using `router.push()`
   - On error: if error message contains "already exists" or similar, show "An account with this email already exists. Sign in instead." (per Copywriting Contract)
   - Generic server error fallback: "Something went wrong. Please try again in a moment."

**2. `app/(auth)/sign-in/page.tsx`** — Sign In page:
   - AuthCard with title "Sign in to your account" (Heading typography: 24px/600)
   - No Display headline (focal point is the card itself per UI-SPEC)
   - Form fields: Email (email input), Password (PasswordInput component)
   - "Forgot your password?" link below password field, linking to `/forgot-password` (per Copywriting)
   - Primary CTA: Button text "Sign in"
   - Below CTA: Separator with "Or continue with" text
   - GoogleOAuthButton
   - Footer link: "Don't have an account? Create one" linking to `/sign-up`
   - On submit: call `signIn.email({ email, password })` from auth-client
   - On success: redirect to `/` using `router.push()`
   - On error: show "Email or password is incorrect. Try again or reset your password." (per Copywriting)

**3. `app/(auth)/forgot-password/page.tsx`** — Forgot Password page:
   - AuthCard with title "Reset your password"
   - Two states: (a) form state, (b) sent confirmation state
   - Form state: Email input field, Primary CTA "Send reset link" (per Copywriting)
   - On submit: call `forgetPassword({ email, redirectTo: "/reset-password" })` from auth-client
   - On success: switch to sent confirmation state
   - Sent confirmation: heading "Link sent", body text "If {email} has an account, you'll receive a reset link shortly." (per Copywriting)
   - Footer link: "Back to sign in" linking to `/sign-in`

**4. `app/(auth)/reset-password/page.tsx`** — Reset Password page:
   - Reads `token` from URL search params (Better Auth sends `?token=xxx` in the reset email link)
   - Three states: (a) form (token valid), (b) success, (c) error (token expired/invalid)
   - If no token in URL: show error state with "This reset link has expired. Request a new one." and link to `/forgot-password`
   - Form state: New password field (PasswordInput), Confirm password field (PasswordInput)
   - Primary CTA: "Set new password" (per Copywriting)
   - Client-side validation: passwords must match (show error if mismatch)
   - On submit: call `resetPassword({ newPassword, token })` from auth-client
   - On success: redirect to `/sign-in` with a success toast/message
   - On error: show "This reset link has expired. Request a new one." (per Copywriting)

**5. `app/(auth)/verify-email/page.tsx`** — Email Verification Pending page:
   - Reads `email` from URL search params (passed from sign-up redirect)
   - AuthCard with title "Check your inbox" (per Copywriting)
   - Body: "We sent a verification link to {email}. Click it to activate your account." (per Copywriting)
   - Note: "Verification is required before accessing paid features." (per Copywriting)
   - Resend button: "Didn't get it? Resend email" (per Copywriting)
   - On resend click: call `sendVerificationEmail({ email })` from auth-client
   - After resend: show confirmation message with `role="status"` and move focus to it (per UI-SPEC Focus Management)
   - Footer link: "Back to sign in" linking to `/sign-in`
