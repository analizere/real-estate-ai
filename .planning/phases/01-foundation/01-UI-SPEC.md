## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.15 |

**Font family:** Geist Sans (`var(--font-geist-sans)`) — already set on `body` in `globals.css`.

**Usage notes:**
- Body (16px/400): form helper text, paragraph content, subscription status descriptions
- Label (14px/400): form field labels, nav items, card metadata, badge text — distinguished from Body by size (14px vs 16px)
- Heading (24px/600): auth form titles ("Sign in to your account"), page section headings
- Display (28px/600): reserved for marketing headline on the sign-up page only ("Analyze any deal in 60 seconds") — distinguished from Heading by size (28px vs 24px)

**Weights used:** 400 (regular), 600 (semibold). Two weights only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (light) / `#0a0a0a` (dark) | Page background, auth card background, form input background |
| Secondary (30%) | `#f5f5f4` (light) / `#171717` (dark) | Nav bar, settings sidebar, subscription status card, divider backgrounds |
| Accent (10%) | `#006aff` | Reserved elements only — see list below |
| Destructive | `#dc2626` | Cancel subscription button and confirmation dialog action only |

**Accent `#006aff` is reserved for:**
- Primary CTA buttons ("Create account", "Sign in", "Upgrade to Pro")
- Google OAuth button border and icon tint
- Active focus ring (`ring-[#006aff]/25`)
- Hyperlinks within body copy (password reset link, "Forgot password?", "Already have an account?")

**Accent is NOT used on:** form field borders (default state), card borders, nav items, secondary buttons, metadata text.

**Border colors:**
- Default input border: `#d1d1d1`

- Card/section border: `rgba(0,0,0,0.06)` (light), `rgba(255,255,255,0.08)` (dark)

**Text colors:**
- Primary text: `#1a1a1a` (light) / `#ededed` (dark) — source: existing `globals.css`
- Muted/helper text: `#535353` (light) / `#a3a3a3` (dark)

## Layout Contract

**Auth pages (sign-up, sign-in, forgot-password, reset-password, verify-email):**
- Centered single-column layout, max-width 400px, vertically centered on page
- White card with 32px internal padding (desktop), 24px padding on mobile
- Platform wordmark + tagline above the card
- No navigation header on auth pages
- Focal point — sign-up: Display headline ("Analyze any deal in 60 seconds.") positioned above the auth card is the primary visual anchor; the card CTA button is the secondary anchor
- Focal point — sign-in, forgot-password, reset-password, verify-email: The auth card itself is the sole visual anchor; no competing Display headline on these screens

**Account settings page:**
- Two-column layout on desktop: 240px left nav + flex-1 content area
- Single-column stacked on mobile (nav collapses to tab row)
- Max content width: 640px
- Focal point — account settings: The subscription status card (top of the content area) is the primary visual anchor; it renders above all other settings sections

## Interaction Contract

**Form validation:**
- Validate on submit, not on every keystroke
- Show field-level error below the input with `role="alert"` and `aria-describedby` linking input to error
- Clear field error on the next input change (not on focus)

**Password input:**
- Show/hide toggle button (eye icon) — right-aligned inside the input
- `type="password"` default, toggled to `type="text"`

**Loading states:**
- Disable the primary CTA button and show a spinner (16px, inside the button) during async operations
- Do not disable or clear form inputs during loading

**Focus management:**
- After sign-in success: redirect — no focus management needed
- After email resend: focus moves to the confirmation message (`role="status"`)
- After cancel subscription dialog opens: focus traps inside the dialog (Radix Dialog handles this)
- After cancel confirmed: focus returns to account settings page, subscription status card updates

**Touch targets:**
- All interactive elements: minimum 44px height (per touch target exception in spacing scale)
- Spacing between form fields: 16px (md token) to prevent mis-taps on mobile
